# LexDesk — CRM-прототип для юриста

Полноценный прототип legal CRM для тестового задания: клиенты, статусы, обзор, воронка, карточка дела, чеклист, история, экспорт.

**Live:** https://elenasamanchuk.github.io/lawyer-crm/

## Запуск

```bash
npm install
npm run dev
```

## Функционал

### Базовый сценарий (ТЗ)
- Таблица клиентов
- Добавление клиента: имя, телефон, статус
- Смена статуса: Новый → В работе → Закрыт
- Счётчики по каждому статусу

### Расширенный функционал
- **Обзор** — дашборд: сегодня к контакту, просроченные, долго без связи
- **Клиенты** — поиск, фильтры, сортировка, приоритет, email, заметки
- **Воронка дел** — drag & drop между статусами
- **Карточка клиента** — drawer с контактами, чеклистом по типу дела, заметкой, историей
- **Валидация полей** — телефон, email, даты, длина заметки
- **Напоминания** — WhatsApp / Telegram с готовым шаблоном
- **Уведомления** — браузерные push при добавлении и смене статуса
- **Экспорт CSV** — выгрузка базы клиентов
- **История действий** — лог по каждому клиенту и общая лента на обзоре

## Стек

React + Vite + TypeScript + Tailwind CSS, данные в `localStorage`.

Паттерны взяты из legal CRM (Clio / MyCase): обзор, pipeline, карточка дела, чеклист, activity log.

## Лог для проверки

**Что делала сама:** сценарий для юриста, приоритеты фич, тексты шаблонов, структура полей, проверка UX на мобильном, деплой.

**Что сделал AI (Cursor):** React-архитектура, компоненты, валидация, Tailwind UI, drag-and-drop воронка, drawer/modal, экспорт CSV, GitHub Pages workflow.

**Почему этот стек:** быстрый рабочий прототип без бэкенда; `localStorage` достаточен для демо; Tailwind + готовые legal CRM-паттерны дают профессиональный UI за часы; GitHub Pages — живая ссылка для проверки.
